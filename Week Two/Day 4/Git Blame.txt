Git blame is a Git command used to track who last modified each line of a file, along with when and in which commit the change was made. It is mainly used for debugging, code review, and understanding code history at a line level.

ðŸ”¹ Basic Syntax
git blame <file-name>

ðŸ”¹ How to Use Git Blame (Workflow)

Initialize or clone a Git repository.

Navigate to the repository directory.

Run git blame on a file to see line-by-line history.

Use the commit ID from the output to inspect detailed changes using:

git log -p <commit-id>

ðŸ”¹ Important Git Blame Options

View raw timestamp

git blame -t <file-name>


View a specific range of lines

git blame -L start,end <file-name>


Show author email instead of name

git blame -e <file-name>


View changes not older than a specific time

git blame --since=3.months <file-name>


Filter out older commits

git blame --since=3.months <file-name> | grep -v '^\^'

ðŸ”¹ Understanding the Output

Each line is annotated with its last modifying commit.

Commits starting with ^ indicate changes older than the given time range.

Helps identify who to contact for a particular piece of code.
